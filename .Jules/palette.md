## 2026-04-12 - Documentation as Primary UX
**Learning:** For repositories without a functional frontend (e.g., backend systems or early-stage projects), documentation like the README.md is the primary user interface. Scannable feature lists and clear status indicators are critical for effective developer experience (DX).
**Action:** Always include a "Status" section and emoji-accented "Key Features" list in READMEs to provide immediate clarity and improve scannability.
